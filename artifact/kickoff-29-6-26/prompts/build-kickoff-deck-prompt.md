# Build kickoff deck prompt

Create a **clean, modern, 5-slide kickoff deck** for the Build community launch.

Audience constraints:
- Mixed skill levels: newcomers, intermediate builders, and experts.
- Keep language clear and energizing.
- Avoid jargon-heavy walls of text.

Deck requirements:
1. Cover slide: Build mission in one sentence.
2. What Build is: "learn by shipping" model and end-to-end startup flow.
3. Core goals: learn with AI agents, teach each other, ship real MVPs.
4. First pilot: Quest Board (not Johnny), with explicit invitation to all skill levels.
5. How to get involved: repository entry points and first actions.

Design requirements:
- 16:9 layout.
- Minimal color palette, strong typography hierarchy.
- High contrast, generous whitespace, consistent footer labels.
- Suitable for a live kickoff event.

Source context to reflect:
- Build README, TEAM, TOOLS, METHODOLOGY docs.
- Build is open source and PR-first.

Output:
- A print-ready PDF.
