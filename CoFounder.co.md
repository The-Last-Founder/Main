# Cofounder.co Setup for Build

Build is the platform.  
Johnny is the first quest.  
Shipping is how we level up.

This document captures how Build will use **Cofounder.co** as part of the community’s work.

Build is not just one project. It is an open-source, learn-by-shipping community that will launch multiple real products with agents.

Therefore, the Cofounder.co account should be set up for **Build as the parent project**, not only for Johnny.

Johnny can be the first quest inside Build, but the account should support the larger structure from day one.

## Why Build-level, not Johnny-level?

We want Cofounder.co to become part of Build’s operating system.

Build will likely include:

- Multiple quests
- Multiple contributors
- Project-specific teams
- Shared learnings across projects
- Reusable specs, prompts, workflows, and product patterns

If we create the account only for Johnny, we may accidentally frame Build too narrowly.

If we create it for Build, Johnny can still get full focus as the first quest, while leaving room for the community to grow.
