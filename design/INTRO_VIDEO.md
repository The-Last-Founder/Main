# Build intro video brief

## Recommendation

**Ideal length:** 1:45 to 2:30 (105-150 seconds).

That is long enough to explain:

1. what Build is,
2. why it feels different,
3. the first quest,
4. how to join.

It is also short enough to stay punchy for social sharing while leaving room for a slightly fuller version for events or deck companions.

**CTA recommendation:** make **Join via WhatsApp** the primary final call to action, with the repo URL shown as a secondary line on the end card.

Why:

- WhatsApp is the fastest path into the community.
- The repo is still important, but works better as the next step after joining.
- A single primary CTA keeps the ending cleaner and more memorable.

**Tone:** fun, lightly MMO/gaming-inspired, but grounded and serious.  
Think: clean UI overlays, quest-map energy, party formation, GitHub receipts.

## Option 1 — "Join the party"

**Target length:** ~105 seconds

**Best for:** homepage, pinned social post, WhatsApp/community invite sharing

### Structure

**0:00-0:10 — Hook**  
Visuals: fast cuts of GitHub issues, pull requests, builder chats, and a quest-map style interface.  
Voiceover:

> Most communities talk about building with AI.  
> Build is where we actually ship.

**0:10-0:28 — What Build is**  
Visuals: Build logo/title, subtle game-party framing, people joining a shared quest board.  
Voiceover:

> Build is an open-source learn-by-shipping community for people building with agents.  
> We pick a real startup idea, build it together in public, and learn what actually works.

**0:28-0:46 — Why it matters**  
Visuals: the flow "Choose a pilot -> Spec -> Code -> Deploy -> Market -> Learn".  
Voiceover:

> Not theory-only meetups.  
> Not fake demos.  
> Real specs. Real pull requests. Real users. Real lessons.

**0:46-1:06 — The first quest**  
Visuals: Quest Board callout, GitHub-native UI, humans plus agents coordinating work.  
Voiceover:

> Our first major quest is Quest Board: a GitHub-native place where humans and agents can propose, improve, rank, adopt, build, and launch open-source projects together.

**1:06-1:26 — Who it is for**  
Visuals: beginner, designer, engineer, writer, operator, researcher, all entering the same party.  
Voiceover:

> You don't need to be an expert.  
> If you can build, write, test, design, research, organize, or help people ship, there is a place for you here.

**1:26-1:45 — CTA**  
Visuals: party fills, one open slot remains, WhatsApp icon/button becomes the main focus, repo URL appears below.  
Voiceover:

> If you want to learn fast by building something real, join the party.  
> Join Build on WhatsApp, then check out the repo and pick your first quest.

### End card

**Headline:** Join Build  
**Primary CTA:** Join via WhatsApp  
**Secondary line:** github.com/The-Last-Founder/Build

## Option 2 — "The open-source builder MMO"

**Target length:** ~150 seconds

**Best for:** launch event, community deck companion, more cinematic explainer

### Structure

**0:00-0:15 — Cold open**  
Visuals: empty digital world / quest map / issue board, then people and agents begin to populate it.  
Voiceover:

> Imagine an open-source world where projects are quests, teams are parties, and shipping is how you level up.

**0:15-0:40 — Define Build**  
Visuals: text overlays from README and TEAM concepts: world, quests, parties, agents.  
Voiceover:

> That is Build.  
> An open-source space for people who want to learn, build, ship, and launch real projects together using AI-native tools.

**0:40-1:05 — The promise**  
Visuals: collaborative docs, agent tools, pull requests, roadmap fragments becoming a product.  
Voiceover:

> We aren't here just to discuss the future.  
> We are here to use agents, document what works, teach each other, and launch real things in public.

**1:05-1:30 — The first pilot**  
Visuals: Quest Board showcase with spec-first, GitHub-native, open-source framing.  
Voiceover:

> The first major Build quest is Quest Board: a GitHub-native, spec-first board for proposing, improving, ranking, adopting, building, and launching open-source projects.

**1:30-1:52 — How the system works**  
Visuals: quest board to party formation to PR merge to launch announcement.  
Voiceover:

> People gather around a quest.  
> Small parties form.  
> Agents help.  
> Work happens in the repo.  
> The goal is simple: ship, learn, and help others ship too.

**1:52-2:15 — Invitation**  
Visuals: MMO-inspired UI settles into a clean real-world CTA panel.  
Voiceover:

> If you have been looking for a serious but fun place to build with others, this is your invite.  
> Join the WhatsApp group, explore the repo, and come help build the next quest with us.

### End card

**Headline:** Your next quest starts here  
**Primary CTA:** Join the Build WhatsApp  
**Secondary CTA:** Explore the repo

## Visual direction

- Minimal, high-contrast editorial design
- MMO-inspired framing without fantasy clichés
- Clean map/quest/party metaphors
- GitHub-native receipts: issues, PRs, markdown, specs
- Human-first energy, not "AI replaces everyone"

## Editing rules

- Keep cuts brisk: 2-5 seconds per shot
- Use on-screen text sparingly
- Let the CTA breathe for at least 4 seconds
- Prefer one narrator with calm conviction over hype voice
- Music should feel playful and forward-moving, not chaotic

## Best single-line pitch

> Build is an open-source learn-by-shipping community where humans and AI agents build a real startup together in public.
